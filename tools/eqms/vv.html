GUIDANCE Verification & Validation
Verification answers "did we build the device right?" — outputs meet inputs. Validation answers "did we build the right device?" — user needs are met in real-world use.
Regulators separate these deliberately. Verification is technical proof of spec conformance; validation requires production-equivalent units and the intended use environment (or a valid simulation). A common failure: running bench tests and calling it validation.
Protocol checklist
✓Write protocol BEFORE executing — auditors will check the date
✓State pass/fail criteria objectively
✓Trace every test back to a DI, DO, or UN
✓Record executor, witness, and any deviations
✓Attach test report and lock once approved
