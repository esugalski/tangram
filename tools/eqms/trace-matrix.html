GUIDANCE Traceability Matrix
Auditors constantly ask one question: "Where's the proof?" The traceability matrix answers it. Each row follows one thread: User Need → Design Input → Design Output → Verification → Validation. An empty cell in any row is a gap you need to close.
This is the single most common document auditors request during an FDA inspection. A complete, gap-free trace matrix is the clearest possible signal that your design process was controlled.
Reading this page
Gaps highlighted in orange are traceability holes — a requirement with no test, or an output without a design input.
Use the filter bar to view by origin (User Need, Regulatory, Risk Control) or to isolate rows with gaps.
Click any cell to view the full record in context.
Common pitfall
Building the trace matrix after the fact, at submission time. It should be maintained continuously — gaps caught early cost hours to fix; gaps found at submission cost months.
